Implementing a Callback Component

The callback component is called by the PSP after a customer has registered payment information with the PSP. It calls the register method of the PaymentRegistrationService interface which is part of the payment service provided by OSCM. The callback component can, for example, be implemented as a servlet.

The callback component passes the organizationKey, paymentTypeKey, paymentInfoKey, and paymentInfoId parameters, which were provided by the request call, to the register method.
