Integrating a Payment Service Provider with OSCM

OSCM provides a Web service interface (payment service) by which operator organizations can integrate payment service providers (PSP) with it.

Communication Between OSCM and a PSP

The following figure illustrates the communication paths between OSCM and a PSP for the case that a customer needs to specify new payment information with the PSP:


The sequence of request and response calls is as follows:

  1. A client accesses the page in OSCM where customers define their payment information.
  2. OSCM performs a synchronous request call to the PSP via the payment service and the PSP integration adapter. OSCM provides the required data and a URL which is used later in the response call.
  3. The PSP creates a new session and returns the URL of its registration page.
  4. The client is redirected to the registration page of the PSP. On this page, the user can enter the payment information. The PSP stores all data provided with the request call, so that they can be returned with the response call. The communication for registering with the PSP and providing the payment information is performed directly between the client and the PSP without involving OSCM or the PSP integration adapter.
  5. The response call from the PSP to OSCM is carried out via the callback component which passes the PSP-specific data to OSCM. This call must be received on port 80 (HTTP) or 443 (HTTPS). The call may include parameters which are not interpreted by the PSP, for example, the customer organization ID. These parameters are provided by OSCM with the request call.
  6. OSCM evaluates the result of the registration and returns a URL to the PSP for redirecting the client to a success or a failure page.
  7. The PSP passes this URL to the client and redirects the client to the page. The page only displays a message indicating whether the operation succeeded or failed. This has no effects on current OSCM sessions.

Overview of PSP Integration Tasks

Integrating a payment service provider with OSCM involves the following implementation tasks:

  • Implement a PSP integration adapter
  • Implement a callback component
These steps are described in detail in the following topics:
  • Implementing a PSP Integration Adapter
  • Implementing a Callback Component

The integration of a PSP involves additional tasks which must be performed by an operator. For details, refer to the Operator's Guide.
