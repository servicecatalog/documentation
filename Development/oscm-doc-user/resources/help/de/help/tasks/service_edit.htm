Service ändern

Auf der Seite Service ändern können Sie alle Eigenschaften eines marktfähigen Service ändern, mit Ausnahme des zugrunde liegenden technischen Service. Im Rahmen der Änderungen haben Sie auch die Möglichkeit, den Servicenamen für Kunden, die Kurzbeschreibung, die Beschreibung und den Namen eines eigenen Tabs in weiteren Sprachen zu hinterlegen.

Sie können einen Service nur ändern, wenn er deaktiviert ist. Laufende Abonnements Ihrer Kunden sind von Ihren Änderungen nicht betroffen. Die Änderungen wirken sich nur auf neue Abonnements aus. Sie müssen den Service aktivieren, damit er den Kunden zum Abonnieren angeboten wird.

Wenn Sie bereits Berechtigungen zum Wiederverkauf des Service erteilt haben, können die Vermittler und Wiederverkäufer sowie ihre Kunden alle von Ihnen gemachten Änderungen sofort sehen, selbst dann, wenn Sie den Service noch nicht aktiviert haben. Laufende Abonnements von Services sind von Ihren Änderungen nicht betroffen. Die Änderungen wirken sich nur auf neue Abonnements aus.

So ändern Sie die Elemente und Eigenschaften eines Service:

  1. Wählen Sie im Feld Service-ID den zu bearbeitenden Service.

    Die Liste enthält alle Services, die Sie definiert haben.

  2. Sie können die folgenden Elemente und Eigenschaften ändern:
    • Service-ID: ID des Service. Sie muss innerhalb der marktfähigen Services Ihrer Organisation eindeutig sein.

      Service-IDs dürfen maximal 40 Zeichen lang sein. Die folgenden Zeichen sind nicht erlaubt:

      ! " # $ % & ' * + , / : ; < = > ? \ ^ `

      Die Service-ID wird zum Beispiel verwendet, wenn Sie den Service aktualisieren oder löschen oder wenn Sie Up-/Downgrade-Optionen oder ein Preismodell für den Service festlegen.

    • Bild für Service: Datei mit einem Bild zur Kennzeichnung des Service.

      Klicken Sie das Eingabefeld Bild hochladen oder Durchsuchen und wählen Sie die Bilddatei aus. Die folgenden Dateiformate werden unterstützt: GIF, JPG und PNG. Das Bild muss 80 x 80 Pixel groß sein.

      Das Bild wird im Servicekatalog auf einem Marktplatz angezeigt, wenn Sie den Service veröffentlichen. Es ist hilfreich für Kunden, wenn diese einen Service suchen.

    • Servicename für Kunden: Aussagekräftiger Name für den Service. Dieser Name sollte einen hohen Wiedererkennungswert für Kunden haben.

      Der Servicename für Kunden wird im Servicekatalog auf einem Marktplatz angezeigt, wenn Sie den Service veröffentlichen. Wenn Sie ihn nicht angeben, wird stattdessen ein Text angezeigt, der besagt, dass der Servicename nicht definiert ist.

    • Kurzbeschreibung: Kurze Beschreibung des Service. Die Kurzbeschreibung darf maximal 256 Zeichen lang sein.

      Die Kurzbeschreibung wird im Servicekatalog auf einem Marktplatz angezeigt, wenn Sie den Service veröffentlichen. Die Kurzbeschreibung liefert in der Regel einen Überblick über die Funktionalität des Service

    • Beschreibung: Ausführliche Beschreibung der Funktionalität des Service.

      Die Beschreibung wird auf einem Marktplatz angezeigt, wenn Kunden Details zu dem Service anfordern.

    Hinweis: Verwenden Sie ausschließlich die Funktionen des Editors, um Textformatierungen vorzunehmen. Direkte HTML-Codierungen im Text sowie Verweise auf externe HTML-Stylesheets oder -Inhalte können zu Fehlern im Seitenlayout führen.
  3. Legen Sie mit der Option Benutzer dem Abo automatisch zuordnen fest, ob der Benutzer, der den Service abonniert, automatisch dem Abonnement zugeordnet werden soll.
  4. In der Tabelle Parameter und Optionen können Sie die Abonnement- und Nutzungsparameter für den Service ändern. Geben Sie in der Spalte Wert die neuen Parameterwerte ein. Der Wert muss in dem Bereich liegen, der für den technischen Service definiert ist. Markieren Sie Wählbar für einen Parameter, wenn Sie festlegen möchten, dass der Parameter Kunden beim Abonnieren des Service als Auswahlmöglichkeit angeboten werden soll.
  5. Definieren oder ändern Sie optional die URL eines Tools zur Parameterkonfiguration. So können Sie ein externes Tool Ihrer Wahl einbinden und Kunden zum Konfigurieren der Parameter beim Abonnieren des Service zur Verfügung stellen.
    Falls ein solches Tool verwendet wird, wird die Tabelle Parameter und Optionen während des Abonnierens durch dieses Tool ersetzt.
    Hinweis: Achten Sie darauf, dass das Tool keine Sicherheitslücken aufweist. Die Syntax und Semantik der URL werden von der Plattform nicht validiert.
  6. Definieren oder ändern Sie optional die URL eines eigenen Tabs an. So können Sie einen zusätzlichen Tab zur Ansicht Details hinzufügen, der angezeigt wird, wenn ein Benutzer zu einem Abonnement weitere Informationen aufruft. Diese URL kann auf eine Web-Seite oder Web-Applikation leiten, deren Inhalt dann auf dem eigenen Tab angezeigt wird. Insbesondere Benutzer, die IaaS-Services nutzen, sind oftmals an Details zu einem abonnierten Service und dessen Betrieb interessiert. Über einen eigenen Tab können Sie Information, die für den Zugriff auf provisionierte Instanzen erforderlich sind, zur Verfügung stellen, zum Beispiel IP-Adressen, Netzwerkinformationen oder Zugangsdaten.
    Hinweis: Achten Sie darauf, dass die Web-Seite oder Web-Applikation keine Sicherheitslücken aufweist. Die Syntax und Semantik der URL werden von der Plattform nicht validiert.

    Aus Sicherheitsgründen werden Requests zu der Web-Seite oder Web-Applikation von der Plattform signiert. Dazu sind bestimmte Konfigurationseinstellungen in der Plattform erforderlich. Außerdem muss sich zur Verifizierung der Signatur das entsprechende Zertifikat der Plattform im Truststore des Application-Servers befinden, auf dem die Web-Seite oder Web-Applikation läuft. Falls Sie Unterstützung benötigen, wenden Sie sich bitte an Ihren Plattformbetreiber.

    Sie können vordefinierte, eigene Tabs einrichten, die Details zu einem IaaS-Service wie beispielsweise OpenStack oder Amazon Web Services (AWS) zur Ansicht bringen. Auf den eigenen Tabs, werden zu den gebuchten Virtuellen Maschinen (VMs) folgende Informationen angezeit:

    • Kopfzeile mit dem Namen des Abonnements
    • Zugangsinformationen
    • Zu jeder VM:
      • Name der Instanz
      • ID der Instanz
      • Art der Instanz
      • Status der Instanz
      • Öffentliche IP-Adresse
      • Private IP-Adressen

    Die URL zum Zugriff auf einen solchen eigenen Tab muss im folgenden Format eingegeben werden:

    https://<host_fqdn>:8881/<service controller name>/serverInformation.jsf

    <host_fqdn> ist der vollqualifizierte Name oder die IP-Adresse der Plattforminstallation, 8881 der benötigte Port. Die Angabe des Ports entfällt, wenn die Plattform mit Proxy betrieben wird. <service controller name> ist die Context-Root des betreffenden Service-Controllers und kann nicht gändert werden.

    Beispiel:

    https://myhost:8881/oscm-app-openstack/serverInformation.jsf

  7. Definieren oder ändern Sie optional den Name des eigenen Tabs an. Dieser Name wird im Titel des Tabs angezeigt. Er darf maximal 40 alphanumerische Zeichen und keine Sonderzeichen enthalten.
  8. Schließen Sie Ihre Änderungen mit Speichern ab.
  9. Um den Servicenamen für den Kunden, die Kurzbeschreibung, die Beschreibung und den Namen eines eigenen Tabs in einer anderen Sprache zu hinterlegen, klicken Sie Lokalisieren. Auf der Seite Servicebeschreibung lokalisieren können Sie anschließend die Texte in allen unterstützten Sprachen eingeben.

Sobald Sie den Service aktivieren, stehen Ihren Kunden die Änderungen für neue Abonnements zur Verfügung. Für Vermittler und Wiederverkäufer sowie deren Kunden sind Ihre Änderungen sofort sichtbar.
