Service definieren

Auf der Seite Service defininieren können Sie einen Service anlegen, den Sie Ihren Kunden anbieten möchten. Sie können den Service Ihren Kunden selbst anbieten. Sie können aber auch Vermittlern oder Wiederverkäufern die Berechtigung zum Wiederverkauf des Service erteilen.

Um einen Service zu definieren, müssen Sie seine Parameter, Bedingungen und Einschränkungen festlegen.

So definieren Sie einen Service:

  1. Wählen Sie den technischen Service, der die Grundlage für den neuen Service sein soll.

    Die Beschreibung des technischen Service wird angezeigt.

  2. Geben Sie die folgenden Informationen für den neuen Service an:
    • Service-ID: ID des Service. Sie muss innerhalb der marktfähigen Services Ihrer Organisation eindeutig sein.

      Service-IDs dürfen maximal 40 Zeichen lang sein. Die folgenden Zeichen sind nicht erlaubt:

      ! " # $ % & ' * + , / : ; < = > ? \ ^ `

      Die Service-ID wird zum Beispiel verwendet, wenn Sie den Service aktualisieren oder löschen oder wenn Sie Up-/Downgrade-Optionen oder ein Preismodell für den Service festlegen.

    • Bild für Service: Datei mit einer Grafik zur Kennzeichnung des Service.

      Klicken Sie das Eingabefeld Bild hochladen oder Durchsuchen und wählen Sie die Bilddatei aus. Die folgenden Dateiformate werden unterstützt: GIF, JPG und PNG. Das Bild muss 80 x 80 Pixel groß sein.

      Das Bild wird im Servicekatalog auf einem Marktplatz angezeigt, wenn Sie den Service veröffentlichen. Es ist hilfreich für Kunden, wenn diese einen Service suchen.

    • Servicename für Kunden: Aussagekräftiger Name für den Service. Dieser Name sollte einen hohen Wiedererkennungswert für Kunden haben. Das Feld ist mit dem Namen des technischen Service vorbelegt, der dem Service zugrunde liegt. Sie können den Namen nach Bedarf ändern.

      Der Servicename für Kunden wird im Servicekatalog auf einem Marktplatz angezeigt, wenn Sie den Service veröffentlichen. Wenn Sie ihn nicht angeben, wird stattdessen ein Text angezeigt, der besagt, dass der Servicename nicht definiert ist.

      Wenn Sie später die Servicedefinition aktualisieren, können Sie den Servicenamen für Kunden in weiteren Sprachen hinterlegen.

    • Kurzbeschreibung: Kurze Beschreibung des Service. Die Kurzbeschreibung darf maximal 256 Zeichen lang sein.

      Die Kurzbeschreibung wird im Servicekatalog auf einem Marktplatz angezeigt, wenn Sie den Service veröffentlichen. Die Kurzbeschreibung liefert in der Regel einen Überblick über die Funktionalität des Service.

      Wenn Sie später die Servicedefinition aktualisieren, können Sie die Kurzbeschreibung in weiteren Sprachen hinterlegen.

    • Beschreibung: Ausführliche Beschreibung der Funktionalität des Service. Das Feld ist mit der Beschreibung des technischen Service vorbelegt, der dem Service zugrunde liegt. Sie können die Beschreibung nach Bedarf ändern.

      Die Beschreibung wird auf einem Marktplatz angezeigt, wenn Kunden Details zu dem Service anfordern.

      Wenn Sie später die Servicedefinition aktualisieren, können Sie die Beschreibung in weiteren Sprachen hinterlegen.

    Hinweis: Verwenden Sie ausschließlich die Funktionen des Editors, um Textformatierungen vorzunehmen. Direkte HTML-Codierungen im Text sowie Verweise auf externe HTML-Stylesheets oder -Inhalte können zu Fehlern im Seitenlayout führen.
  3. Wenn der Benutzer, der den Service abonniert, dem Abonnement automatisch zugeordnet werden soll, wählen Sie die Option Benutzer dem Abo automatisch zuordnen. Standardmäßig muss jeder Benutzer, einschließlich demjenigen, der den Service abonniert, dem Abonnement explizit zugeordnet werden, bevor er den Service nutzen kann.
  4. Geben Sie in der Tabelle Parameter und Optionen Werte für die Abonnement- und Nutzungsparameter ein:
    • Abonnementparameter legen zum Beispiel die maximale Anzahl der Benutzer pro Abonnement oder die maximale Laufzeit eines Abonnements fest. Nutzungsparameter bestimmen zum Beispiel das Freischalten von Funktionen der zugrunde liegenden Anwendung oder die maximale Anzahl an Ordnern, Dateien oder Objekten, die angelegt werden können. Sowohl Abonnement- als auch Nutzungsparameter sind durch den zugrunde liegenden technischen Service vorgegeben.
    • In der Spalte Beschreibung sehen Sie eine kurze Beschreibung jedes Parameters. Diese Beschreibung wird vom Hersteller geliefert.
    • Markieren Sie Wählbar für einen Parameter, wenn Sie festlegen möchten, dass der Parameter Kunden beim Abonnieren des Service als Auswahlmöglichkeit angeboten werden soll.
    • In der Spalte Wert können Sie für jeden Parameter einen Wert festlegen. Der Wert muss in dem Bereich liegen, der für den technischen Service definiert ist.
  5. Geben Sie optional die URL eines Tools zur Parameterkonfiguration an. So können Sie ein externes Tool Ihrer Wahl einbinden und Kunden zum Konfigurieren der Parameter beim Abonnieren des Service zur Verfügung stellen.
    Falls ein solches Tool verwendet wird, wird die Tabelle Parameter und Optionen während des Abonnierens durch dieses Tool ersetzt.
    Hinweis: Achten Sie darauf, dass das Tool keine Sicherheitslücken aufweist. Die Syntax und Semantik der URL werden von der Plattform nicht validiert.
  6. Geben Sie optional die URL eines eigenen Tabs an. So können Sie einen zusätzlichen Tab zur Ansicht Details hinzufügen, der angezeigt wird, wenn ein Benutzer zu einem Abonnement weitere Informationen aufruft. Diese URL kann auf eine Web-Seite oder Web-Applikation leiten, deren Inhalt dann auf dem eigenen Tab angezeigt wird. Insbesondere Benutzer, die IaaS-Services nutzen, sind oftmals an Details zu einem abonnierten Service und dessen Betrieb interessiert. Über einen eigenen Tab können Sie Information, die für den Zugriff auf provisionierte Instanzen erforderlich sind, zur Verfügung stellen, zum Beispiel IP-Adressen, Netzwerkinformationen oder Zugangsdaten.
    Hinweis: Achten Sie darauf, dass die URL keine Sicherheitslücken aufweist. Die Syntax und Semantik der URL werden von der Plattform nicht validiert.

    Sie können vordefinierte, eigene Tabs einrichten, die Details zu einem IaaS-Service wie beispielsweise OpenStack oder Amazon Web Services (AWS) zur Ansicht bringen. Auf den eigenen Tabs, werden zu den gebuchten Virtuellen Maschinen (VMs) folgende Informationen angezeit:

    • Kopfzeile mit dem Namen des Abonnements
    • Zugangsinformationen
    • Zu jeder VM:
      • Name der Instanz
      • ID der Instanz
      • Art der Instanz
      • Status der Instanz
      • Öffentliche IP-Adresse
      • Private IP-Adressen

    Die URL zum Zugriff auf einen solchen eigenen Tab muss im folgenden Format eingegeben werden:

    http://<app-host-name>:<app-port>/<service controller name>/serverInformation.jsf

    oder, abhängig vom verwendeten Protokoll:

    https://<app-host-name>:<app-port>/<service controller name>/serverInformation.jsf

    Beispiel:

    https://MyAPPHost:8880/oscm-app-openstack/serverInformation.jsf

  7. Geben Sie optional den Name des eigenen Tabs an. Dieser Name wird im Titel des Tabs angezeigt. Er darf maximal 40 alphanumerische Zeichen und keine Sonderzeichen enthalten.
  8. Schließen Sie Ihre Eingaben mit Speichern ab.

Der Service wird angelegt und kann weiter bearbeitet werden. Sie müssen als nächstes ein Preismodell definieren und den Marktplatz festlegen, auf dem Sie den Service veröffentlichen möchten. Anschließend können Sie den Service aktivieren, damit er von Kunden abonniert werden kann.

Sie können nach Bedarf Up- und Downgrade-Optionen für den Service festlegen. Außerdem können Sie Vermittlern und Wiederverkäufern die Berechtigung zum Wiederverkauf des Service erteilen.
