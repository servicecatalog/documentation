Customizing the marketplace layout

On the Customize layout page, you can specify a style sheet that defines the branding and layout of a marketplace, i.e. the colors, fonts, background images, and logos. The best way to create an appropriate style sheet is to use the platform's default style sheet as a basis and adapt it as required. You then provide the customized style sheet and the files it references to the platform operator so that he can make them available in the oscm-branding container. When the operator is finished, he can provide the URL to the style sheet that can then be specified on the Customize layout page. If no URL is set, the default style sheet is used.

The platform's default style sheet (mp.css) and the files and images it references are provided in the branding package, a ZIP file which you can download on the Customize layout page. The branding package also includes sample HTML pages of a marketplace. You can use these HTML pages to preview and test your customizations. In the style sheet, you can find comments and descriptions of the relevant elements of the HTML pages as well as information on which elements you must not customize in order to ensure that the page layout does not get broken.

Be aware of the following:

  • Your organization is responsible for providing the correct version of the customized style sheet and all referenced files to the platform operator.
  • If the server hosting the style sheet cannot be reached, the behavior depends on the Web browser used to access the marketplace. For example, the browser could freeze or show the content without any style.

    In order to reduce this risk to a minimum, the platform operator must ensure that the customized style sheet is stored in a location that is always available.

  • When you save your style sheet settings on the Customize layout page, the platform checks whether the server hosting the style sheet is available and that text or CSS content is provided when accessing the URL. Be aware, however, that the platform does not check for any files and images referenced by the style sheet.
  • A new layout becomes effective as soon as the platform operator has loaded your changes to the system. They are then effective for a user the next time he logs in. Anonymous users need to close and reopen their Web browser. Thus, make sure that the previous style sheet is still available for some time when you update it to a newer version.
  • You need to ensure that any Web browser accessing your marketplace is forced to retrieve the style sheet and the referenced files from the server when a new version is available and accessible from the Internet.

    You can achieve this by adding version information to the style sheet URL.

    Every time you provide a new version of the style sheet, you change the version information and thus the URL.

    Another way is to use entity tags (ETags), which are part of HTTP. ETags are a mechanism that HTTP provides for Web cache validation. They allow a client to make conditional requests. A Web server does not need to send a full response if the content has not changed.

The branding of a marketplace also comprises the stage and the texts (labels of user interface elements, email message texts, etc.). The stage, i.e. the area where you can place advertisements, eye catchers, or further information, can be customized using the Customize stage menu option in the Marketplace menu. The texts can be customized using the Customize texts menu option in the Marketplace menu.

To customize the layout of a marketplace:

  1. Download the branding package by clicking the corresponding button.

    The branding package is a ZIP file. It contains the platform's default style sheet (mp.css), the files and images it references, as well as HTML pages for previewing and testing your adaptations to the style sheet.

  2. Extract the ZIP file to any location in your environment.
  3. Adapt the style sheet as required.

    To test your customizations, open one or several of the sample HTML pages. Refresh the browser content after any change you make.

    Be aware that you are responsible for the correct page layout. This means that you should always thoroughly check the layout before publishing the customized style sheet.

  4. Provide the customized style sheet and the referenced files, images, and fonts to the platform operator (as a ZIP or TAR archive), and ask him to load your changes into the system.
  5. When you are notified that he is finished, go back to the Customize layout page.
  6. From the Marketplace list, select the marketplace for which you want to customize the layout.

    The list contains all marketplaces owned by your organization.

  7. In the Style sheet URL field, enter the URL pointing to your customized style sheet.

    Example:

    http://<your_host>/Version1.1/CustomizedStyleSheet/mp.css

  8. Click Save to make the new layout and branding of your marketplace available.

    The platform checks whether the server hosting the style sheet is available and that text or CSS content is provided when accessing the URL. If this is not the case, an error message is displayed and the URL is not saved. Be aware, however, that the platform does not check for any files and images referenced by the style sheet.

    A new layout becomes effective for a user the next time he logs in. Anonymous users need to close and reopen their Web browser.

    To restore the default layout, delete the entry from the Style sheet URL field and click Save.
