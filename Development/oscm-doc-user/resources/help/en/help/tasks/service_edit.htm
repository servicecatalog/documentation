Updating a service

On the Update service page, you can change all elements of a marketable service definition except for the technical service on which the marketable service is based. Updating a service also allows you to provide the service name for customers, the short description, the description, and the name of the custom tab in different languages.

You can only update a service that is deactivated. Existing subscriptions of your customers are not affected by the update. Any changes you make are effective for new subscriptions only. To provide the service to customers for subscription, you must activate it.

If you have granted resale permissions, the brokers and resellers as well as their customers can immediately view any changes you make to the service definition, even if you have not yet activated your service. Existing subscriptions to services are not affected by the update. Any changes you make are effective for new subscriptions only.

To change the elements and properties of a service definition:

  1. From the Service ID list, select the marketable service you want to change.

    The list contains all services you have defined.

  2. You can change the following elements and properties:
    • Service ID: Identifier of the service. It must be unique within the marketable services of your organization.

      The service ID is restricted to 40 characters and must not contain any of the following characters:

      ! " # $ % & ' * + , / : ; < = > ? \ ^ `

      The service ID is offered for selection, for example, when updating or deleting services, or when specifying upgrade and downgrade options or a price model.

    • Service image: File containing the image for identifying the service.

      Click the Image upload field or Browse, and select the file containing the image. Supported file formats are GIF, JPG, and PNG. The size of the service image must be 80 x 80 pixels.

      The service image is shown in the service catalog on a marketplace when you publish the service. It may be useful, for example, for customers to identify the service.

    • Service name for customers: Meaningful name for the service. It should have a high recognition value for customers.

      The service name for customers is shown in the service catalog on a marketplace when you publish the service. If you do not specify it, a string is displayed saying that the service name is not defined.

    • Short description: Brief description of the service. The length of the short description is restricted to 256 characters.

      The short description is shown in the service catalog on a marketplace when you publish the service. Usually, it contains an overview of the features the service provides.

    • Description: Detailed description of the features offered by the service.

      The service description is displayed on a marketplace when customers request details of the service.

      Note: Make sure to use only the controls provided in the editor for formatting your text. Manually entered HTML tags and references to external style sheets or content may result in errors in the page layout.
  3. By selecting or deselecting Auto-assign the subscribing user, you can determine whether a user subscribing to this service is automatically assigned to the subscription after having completed the subscription process.
  4. In the Service parameters table, you can change the subscription and usage parameters for the service by editing the values in the Value column. The value must be within the range defined for the technical service. By selecting the User option check box for a parameter you can specify whether it is offered as an option to customers who subscribe to the service.
  5. Optionally, specify or change the URL of a parameter configuration tool. In this way, you can link an external tool of your choice. Your customers can use this tool for configuring the subscription and usage parameters when they subscribe to the service.
    If an external tool is used, the Service parameters table for configuring the parameters is replaced by the tool's interface during the subscription process.
    Note: Make sure that the URL pointing to the Web page or Web application for the custom tab is secure. The URL is not validated by the platform, neither for syntax nor for semantic.

    For secure communication, the requests sent to the Web page or Web application are signed by the platform. For this to be possible, specific configuration settings in the platform are required. The application server used for the Web page or Web application needs the corresponding certificate of the platform in its truststore for verifying the signature. Contact your operator for support, if required.

  6. Optionally, specify or change the URL of a custom tab. In this way, you can add a tab to the Details view for subscriptions on the marketplace. This URL may point to a Web page or Web application the content of which will be retrieved and shown on the custom tab. Particularly users of IaaS services may be interested in additional details on the service and the operation of the service they subscribed to. Using a custom tab, you can provide the information needed for accessing provisioned instances, such as IP addresses, network information, or user credentials.
    Note: Make sure that the URL pointing to the Web page or Web application for the custom tab is secure. The URL is not validated by the platform, neither for syntax nor for semantic.

    Predefined custom tabs are available for viewing details on subscriptions to an IaaS service such as OpenStack or Amazon Web Services (AWS). On the custom tab, the booked Virtual Machines (VMs) are listed with the following information:

    • Header with the name of the subscription
    • Access information
    • For each booked VM:
      • Instance name
      • Instance ID
      • Instance type
      • Instance status
      • Public IP address
      • Private IP addresses

    You can set the URL for accessing the custom tab in the following format:

    https://<host_fqdn>:8881/<service controller name>/serverInformation.jsf

    <host_fqdn> is the fully qualified name or IP address of the host used to access the platform, 8881 is the port of APP. Omit the port if the platform is operated with its proxy. <service controller name> is the default context root of the relevant service controller and cannot be changed.

    For example:

    https://myhost:8881/oscm-app-openstack/serverInformation.jsf

  7. Optionally, specify or change the Name of the custom tab as shown to your customers. The name is restricted to 40 alphanumeric characters and must not contain special characters.
  8. To save your changes, click Save.
  9. To provide the service name for customers, the short description, the description, and the custom tab name in another language, click Localize. The Localize service page allows you to provide the elements describing the service name in all supported languages.

After activating the service, the changes are made available to your customers for new subscriptions. Brokers and resellers as well as their customers can immediately view your changes.
